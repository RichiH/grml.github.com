How to contribute to Grml

We want you!

Grml is an active open source project, available since 2003 and serving tens of thousands of users.

We need your help! Driven by a small team in their free time we can't get as much stuff done as we'd like to. If Grml saved your ass and want to give something back to the Grml project this website will get you started.

Disclaimer Don't be afraid of us. You might think that you are not experienced enough to contribute to Grml (or any other open source project). But let us tell your otherwise: by using Grml and hitting this page you've pretty good chances to already provide a decent baseline matching our needs. No matter how small your contribution might look to you, it really matters for us! Don't be shy, let's make Grml a kick ass solution!

How to contribute to Grml.org

Grml joined the Github party, all our repositories are available at github.com/grml. If you already have a Github account it's rocking simple to contribute.

Notice: while it brings benefit to both sides of the party there really is no need to have an Github account. We even run a copy of all the Github stuff on our own infrastructure. But social coding rocks!

So, how to get started?

  • You already have a patch? Go straight towards get your work upstream
  • You already have a repository and want to create a patch out of it? Checkout how to create a patch
  • Already know which repository to work with? Skip over to Fork!
  • Not sure where to find the according sources? Just read on...



Identifying the appropriate Git repository

Soooo many Git repositories, you might ask where to start? We are here to serve you!

Grml's project homepage is available as git repository. If you found a typo, feel the desparate need to fix our denglish or work on our documentation that's the right place to start.

If you are also a huge fan of Zsh, Vim, Screen and Tmux and want to pimp their configs the grml-etc-core repository is where the music plays.

You're already enjoying your own Live system based on Grml? Fantastic! Don't forget to check out grml-live.

Now let's visit the machine room. The boot process of Grml starts with the Linux kernel, which includes patches and configuration found in the grml-kernel repository. The initramfs takes over, driven by live-boot-grml. Once the initramfs stage was finished the main bootup process featured by grml-autoconfig takes over. live-boot-grml and grml-autoconfig are also the place where all the nifty Grml boot options are implemented at.

A simple frontend to set up booting via network (PXE) is known as grml-terminalserver.

To get your Live system onto a USB pen grml2usb is waiting for you.

Notice: none of the described repositories matches your needs? Either figure out the repository and try to spot the appropriate repository or just talk to us.




Fork!

You identified the according Git repository? Great! The easiest way to play with the repository is by choosing the 'Fork' feature at the right side of the upper screen:

This should give you your own copy of the Git repository and you can commit to it without even asking us for permission. Then you can simply create a patch.

If you don't have a Github account (yet?) then you can grab the Git repository anyway. On the Github web page you can click on the 'Git Read-Only' option. Then check out the URL (being 'git://github.com/grml/grml-etc-core.git' in the following screenshot):

Check out the repository and explore it:

% git clone git://github.com/grml/grml-etc-core.git
% cd grml-etc-core



Create a patch

Creating a patch is simple. Check out a new branch in your repository (to avoid messing and conflicts possibly showing up in master):

% git checkout -b mygreatfeature

Implement your bugfix/feature and if you're happy about it commit your work (replace $FILES with the files you changed):

% [hack hack hack] % git add $FILES % git commit -v

That's it pretty much. If you aren't familiar with Git yet and want to learn more git-scm.com has plenty of useful information.

Finally it would be great to share your work with us!




Get your work upstream

Getting your work into Grml isn't just about making Grml even hotter. It also has advantages for you! You get your features tested and maintained by a community, gain credits and get immediate builds of it served by our awesome continuous integration lab. Alright!

If you chose to forked the repository all you've to do is click on the 'Pull Request' button on your Github page:

If you're interested in further documentation regarding the pull request head over to the Github page about pull requests.

Don't have a Github account and/or prefer mail? No worries. Just create a patch based on your work:

% git format-patch origin/master

Then send the created patch file(s) to bugs (at) grml.org. Either use your common mail client or if you prefer to use Git itself invoke:

% git send-email 000*



Contact

Important If you're unsure about something (like which repository is the right one, is the bug in question fixed already, what's the best way to get started with your awesome new feature or think this page is incomplete): please talk to us! We're usually available on IRC in #grml on freenode (most of us sitting in the CET timezone). If you prefer asynchronous communication instead subscribe to the grml-devel mailinglist and flood our mailboxes!
